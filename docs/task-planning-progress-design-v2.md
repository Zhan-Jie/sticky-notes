# Sticky Notes 任务规划与执行设计草案 V2（轻量子任务版）

## 1. 文档目的
V2 的目标不是把应用做成“大而全”的项目管理软件，而是解决一个更核心的问题：
- 让日常任务执行更顺手
- 降低规划心智负担
- 在“想到就记、边做边拆”的真实工作流里保持可控

V2 只做一件关键增强：**在当前任务下增加可动态维护的子任务（Subtask）**。

## 2. 问题重定义
你当前遇到的问题，本质不是“功能不够多”，而是“不同类型信息混在一个工具里”。

工作中的信息至少分三类：
1. `执行型`：今天/现在要做的动作，要求快录入、快勾选。
2. `跟踪型`：复杂长期事项的拆解、阶段进度、资料沉淀，要求结构化和可追踪。
3. `知识型`：完成后可复用的经验、指标口径、文档索引、密钥与环境细节等。

V2 明确选择：**Sticky Notes 专注第 1 类（执行型）**，并通过轻量链接能力与第 2、3 类工具协同。

## 3. 产品边界（必须明确）
### 3.1 Sticky Notes（本应用）
- 管理“当前可执行事项”
- 支持主任务下动态子任务
- 支持快速完成标记与最小上下文记录

### 3.2 飞书文档 / Obsidian（外部）
- 管理复杂事项的目标拆解、方案推进、问题跟踪
- 管理过程资料（分支、文档地址、上线清单等）

### 3.3 知识库（外部）
- 管理沉淀后的可复用知识与长期参考信息

### 3.4 非目标（V2 不做）
- 不做多层无限任务树
- 不做甘特图/里程碑/复杂依赖编排
- 不做完整知识库系统
- 不做“自动完成父任务”的强规则推断
- 不做“子任务一键提升为父任务”

## 4. V2 核心模型
### 4.1 实体定义
1. `ParentTask`（父任务）
2. `Subtask`（子任务，仅一层）

说明：
- 子任务只隶属于一个父任务，不再嵌套孙任务。
- 先保证“简单可用”，后续再评估是否扩展深层树。

### 4.2 状态定义
父任务状态（沿用现有能力，不新增枚举）：
- `todo`：未完成（包含“未开始/进行中”，由用户继续推进）
- `done`：完成（仅用户手动触发）
- `suspended`：挂起（沿用现有能力）

父任务显示态（UI 派生，不持久化）：
- `in_progress`：进行中（用于满足“子任务有推进后父任务就是进行中”的感知）。触发条件：`activeSubtaskId` 非空；或存在已完成子任务；或父任务处于计时中/超时中。说明：该显示态不改变父任务 `status` 字段；V2 当前不强制展示独立“进行中”标签。

子任务状态（V2 最小化）：
- `todo`
- `done`

### 4.3 状态规则（关键）
1. 新建父任务默认 `todo`。
2. 子任务可在执行过程中随时新增，不要求预先完整拆解。
3. 任意一个子任务被标记 `done` 后，若父任务为 `todo`，父任务状态保持 `todo`，并进入 `in_progress` 派生态。
4. 父任务为 `suspended` 时，子任务完成/新增不自动把父任务切回 `todo`（避免过度自动化；由用户手动恢复）。
5. 即使所有子任务都 `done`，父任务也**不会自动变为 `done`**，父任务状态仍保持 `todo`。
6. 父任务必须由用户显式点击“完成”才进入 `done`。
7. 父任务已 `done` 后若新增子任务或将任一子任务从 `done` 改回 `todo`，父任务自动回到 `todo`（避免“已完成但仍有未做项”的冲突）。
8. 子任务全部删除后，父任务状态不强制回退，由用户决定（避免过度自动化）。

## 5. 交互与视觉设计
### 5.1 列表层级
1. 父任务作为主行展示（保持当前样式主视觉）。
2. 子任务默认折叠，点击展开显示在父任务下方。
3. 子任务行左侧缩进，形成明确从属关系。

### 5.2 状态标识差异
1. 父任务完成控件：保持现有复选框样式。
2. 子任务完成控件：使用`圆圈`样式（未完成空心圆，完成后带勾/填充），与父任务视觉区分。
3. V2 当前不展示独立“进行中”标签，避免信息噪音。
4. V2 当前不展示“有备注”的右侧小圆点提示。

### 5.3 录入体验
1. 子任务展开后，默认不直接展示输入框。
2. 子任务列表底部展示`添加子任务`按钮，点击后出现输入框。
3. 输入框内回车保存；保存后回到简洁列表态。
4. 不弹复杂表单，不要求填写耗时、优先级、依赖等字段。

### 5.4 进度感知（轻量）
1. 父任务主标题后固定展示子任务进度：`（x/y）`。
2. 当标题过长时，主标题可省略；`（x/y）`不省略、不被覆盖。
3. 该进度仅用于感知推进，不参与“自动完成父任务”。

### 5.5 完成项收纳（保持轻量的关键）
1. 展开子任务时，已完成项展示在前，未完成项展示在后。
2. 默认展示最多 2 个最近已完成子任务（按 `doneAt` 倒序；若无 `doneAt` 则按 `order` 倒序）。
3. 若仍有被隐藏的已完成项，则在列表最前展示一行可点击提示 `已完成（n）`，其中 `n` 为被隐藏数量。
4. 点击展开后显示全部已完成项；提示行仍保留，显示当前展示数量，便于再次收起。
5. 若没有被隐藏的已完成项，则不展示 `已完成（n）` 提示行。
6. `已完成（n）` 前的展开图标与子任务圆圈对齐，保证视觉一致。
7. 当子任务数量明显膨胀（例如 > 20 条）时，提示用户将复杂跟踪迁移到外部文档（本应用仍只保留执行清单与链接）。

### 5.6 当前子任务（轻量上下文）
1. 子任务支持被选为“当前进行中的子任务”（每个父任务最多 1 个，可为空）。
2. 设置为当前：点击子任务行正文区域；点击子任务圆圈仍表示完成/取消完成。
3. 取消当前：再次点击已选中的当前子任务正文区域。
4. 若存在当前子任务：父任务主标题显示当前子任务名称；父任务名称下沉到下一行小字。
5. 若不存在当前子任务：主标题显示父任务名称（默认形态）。
6. 倒计时固定显示在主标题行最右侧；有无当前子任务都可显示倒计时。
7. 不展示 `当前：` 这类冗余前缀。
8. 当前子任务完成后的处理：默认自动切到下一个未完成子任务（按 `order`）；若不存在则清空当前子任务。
9. 当前子任务删除后的处理：默认自动切到下一个未完成子任务（按 `order`）；若不存在则清空当前子任务。

### 5.7 行内显隐与操作反馈
1. 父任务行操作按钮默认隐藏，仅在鼠标悬浮任务行时展示。
2. 子任务行的完成/删除按钮默认隐藏，仅在悬浮该子任务行时展示。
3. 鼠标手形反馈仅出现在可点击控件（子任务圆圈、完成按钮、删除按钮）；子任务正文区域使用普通光标。
4. 子任务名称支持双击编辑。
5. 未完成子任务支持拖动排序（不影响已完成分组）。

### 5.8 已完成任务归档（Markdown 备份）
目标：保持应用轻量（已完成列表不无限增长），同时把“已完成事项”以可读文本形式沉淀到外部。

1. 设置页支持配置“备份目录”（系统文件夹路径）。未配置时默认使用应用数据目录下的 `backup/` 子目录。
2. 自动归档规则：应用内最多保留最近 **10 个**已完成父任务；超过 10 个时，自动将更早的已完成任务导出到 Markdown，并在导出成功后从应用内删除这些已完成任务。触发时机：父任务标记完成后；应用启动时也执行一次归档检查。
3. 手动归档：设置页提供按钮 `立即导出所有已完成任务`，导出成功后清空应用内所有已完成父任务（仅保留未完成与挂起任务）。
4. 可靠性要求：仅在 Markdown 文件写入成功后才删除应用内数据；写入失败则不删除，并给出可感知提示（例如 toast/对话框）。
5. 导出内容（父任务维度）：
   - 父任务文本、完成时间 `doneAt`、上下文 `contextText`
   - 子任务列表（含完成状态），以及 `activeSubtaskId` 对应的“当前事项”提示（若仍存在）
6. 文件组织（建议，保持简单）：
   - 文件路径：`<备份目录>/sticky-notes-done-YYYY-MM.md`
   - 写入方式：追加写入（每次归档追加一个 `## 导出时间：...` 段落），便于长期累计与检索
   - 格式示例：
     - `- [x] <父任务文本> (doneAt: 2026-03-01 10:40)`
     - `  - 子任务:`
     - `    - [x] ...`
     - `    - [ ] ...`
     - `  - 上下文: ...`

## 6. 外部工具协同方法（简版）
为避免应用膨胀，定义一条轻量协同流程：

1. 在 Sticky Notes 建立父任务，执行中动态补子任务。
2. 当任务变复杂（需方案、长记录、多资料）时，在父任务上下文放外部链接（飞书/Obsidian），并保证链接可被快速打开（例如上下文中自动识别 URL 并可点击/一键打开）。
3. 父任务完成时，将可复用结果沉淀到知识库；Sticky Notes 只保留简短摘要或链接。

这套方式的核心是：**执行在 Sticky Notes，复杂跟踪在文档，知识沉淀在知识库。**

## 7. 数据结构草案（面向实现）
```json
{
  "tasks": [
    {
      "id": "task_001",
      "text": "完成测试环境部署与冒烟验证",
      "status": "todo",
      "createdAt": "2026-03-01T10:00:00+08:00",
      "doneAt": null,
      "order": 1,
      "contextText": "外部跟踪文档: https://...",
      "activeSubtaskId": "sub_002",
      "subtasks": [
        {
          "id": "sub_001",
          "taskId": "task_001",
          "text": "修复 docker-compose 启动失败",
          "status": "done",
          "createdAt": "2026-03-01T10:10:00+08:00",
          "doneAt": "2026-03-01T10:40:00+08:00",
          "order": 1
        },
        {
          "id": "sub_002",
          "taskId": "task_001",
          "text": "补齐测试数据初始化脚本",
          "status": "todo",
          "createdAt": "2026-03-01T10:45:00+08:00",
          "doneAt": null,
          "order": 2
        }
      ]
    }
  ]
}
```

字段说明：
- `subtasks` 为父任务内嵌数组（V2 可先不拆表，降低改造成本）。
- 子任务保留独立 `doneAt`，便于后续做简单“今日完成”统计。
- `activeSubtaskId` 表示父任务当前选中的子任务（可为空），用于主标题切换到“当前子任务视角”。
- 父任务 `status` 仍保持 `todo/suspended/done`；“进行中”由 UI 依据子任务进展与计时状态派生展示。

备份设置字段（草案）：
```json
{
  "backupDir": "/home/yourname/Documents/sticky-notes-backup",
  "doneTaskRetention": 10
}
```

说明：
- `backupDir` 为空表示使用默认备份目录（应用数据目录下的 `backup/`）。

## 8. 关键流程
### 8.1 创建与执行
1. 新建父任务（与当前一致）。
2. 开始做事时，想到步骤就添加子任务。
3. 可选：选中某个子任务作为当前进行中事项（主标题切换为子任务名称，父任务名显示为小标题）。
4. 做完一个子任务即点圆圈完成；父任务状态仍保持 `todo`，不自动 `done`。

### 8.2 收尾
1. 子任务即使全部完成，父任务仍保持 `todo`，不自动完成。
2. 用户确认整体目标达成后，手动标记父任务 `done`。

### 8.3 返工
1. 已完成父任务新增子任务或取消任一子任务完成时，父任务从 `done` 自动回到 `todo`。
2. 保留历史完成时间，便于回顾返工成本。

## 9. 迁移与兼容
1. 老数据（无 `subtasks`）默认按空数组处理。
2. 现有 `TaskStatus` 兼容保留，不新增 `doing` 状态。
3. 仅在 UI 层增加子任务区域，不破坏当前任务计时与上下文能力。

## 10. 分期落地建议
### Phase 1（MVP）
1. 数据模型增加 `subtasks`。
2. 任务行支持展开/收起子任务。
3. 子任务新增、完成、删除。
4. 落实父任务状态联动规则（但不自动完成）。
5. 已完成子任务默认折叠收纳，避免子任务列表膨胀影响主界面轻量体验。
6. 子任务支持选中为当前进行中事项。

### Phase 2（易用性增强）
1. 父任务主标题后固定展示 `（x/y）` 子任务进度。
2. 子任务输入框改为按需展开（默认只显示`添加子任务`按钮）。
3. 父任务上下文中增加“外部文档链接”快捷入口。
4. 设置页支持选择备份目录。
5. 已完成父任务自动归档：应用内仅保留最近 10 个已完成任务，超出的自动导出到 Markdown 后删除。
6. 设置页支持 `立即导出所有已完成任务`（导出成功后清空已完成列表）。
7. 子任务支持双击编辑与未完成子任务拖拽排序。
8. 主窗口失焦或切换到其他父任务时，自动收起子任务列表。

### Phase 3（复盘增强）
1. 今日完成子任务数统计。
2. 父任务完成时可一键复制“执行摘要模板”到剪贴板。

## 11. 验收标准
1. 用户可在 3 秒内为任一父任务新增子任务。
2. 父任务操作按钮默认隐藏，鼠标悬浮父任务行时再显示。
3. 所有子任务完成时，父任务不自动完成。
4. 用户可显式手动完成父任务，行为可逆（可改回 `todo`）。
5. 现有无子任务数据可无损读取和保存。
6. 用户可快速将任一子任务设为“当前进行中事项”；有当前子任务时主标题展示子任务、父任务名下沉为小标题。
7. 设置页可配置备份目录；未配置时需有明确行为（例如提示未设置，或使用默认目录并可被用户知晓）。
8. 应用内已完成父任务数量超过 10 个时，自动将更早的已完成任务导出到 Markdown；导出成功后这些任务从应用内删除，最终已完成任务数量回到 10 个。
9. 设置页点击 `立即导出所有已完成任务` 后：生成 Markdown 文件（路径可追溯/可打开），导出成功后应用内已完成任务清空。
10. 已完成子任务分组可展开/收起：折叠态显示被隐藏数量，展开态可再次收起。
11. 主窗口失焦或切换到其他父任务时，子任务列表自动收起。
12. 子任务支持双击编辑；未完成子任务支持拖动排序。
13. 有当前子任务时，主标题显示子任务名、父任务名下沉为小标题；无当前子任务时主标题显示父任务名。
14. 倒计时固定显示在主标题行右侧；不展示 `当前：` 前缀。

## 12. 设计原则总结
1. **轻量优先**：先解决“做事时的摩擦”，而非“建一个全能系统”。
2. **动态拆解**：允许执行中补子任务，符合真实工作节奏。
3. **手动收口**：父任务完成必须由用户确认，避免错误自动化。
4. **工具协同**：复杂跟踪和知识沉淀交给更合适的文档/知识工具。
